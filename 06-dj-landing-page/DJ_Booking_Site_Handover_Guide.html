DJ Booking Site — Project Handover & Technical Guide

Prepared by: Maruwa Thomas · AWS Cloud Engineer · Email: Maruwathomas@gmail.com · GitHub: github.com/Maruwat21

1) Overview

A single-page, responsive landing site for DJs to showcase style, events, and a simple booking flow. Hosted on Amazon S3 (static website hosting) for extremely low monthly cost and high reliability. Region: us-east-1.

2) What’s included

  • Hero header with translucent DJ image and call-to-action
  • Events / gallery section
  • Booking/contact form (mailto or future API)
  • Clean, mobile-first CSS
  • Deployed to S3 static website hosting

4) Deploy / Update Steps (S3)

  1. Open S3 ▸ Bucket: (your bucket name) ▸ Properties ▸ Static website hosting: Enable, Index: index.html.
  2. Upload/replace index.html, styles.css, and assets/.
  3. Ensure public read access via a bucket policy that allows s3:GetObject to arn:aws:s3:::YOUR_BUCKET/*.
  4. Open your S3 website endpoint (e.g., http://YOUR_BUCKET.s3-website-us-east-1.amazonaws.com/) and test.

5) Optional Enhancements

  • Custom domain (Route 53) and HTTPS (CloudFront)
  • Contact email via Amazon SES (API Gateway + Lambda)
  • Payment links (Stripe Checkout) for deposits

7) Estimated AWS Monthly Cost

  • S3 static site (low traffic): ~ $0.50 – $2.00 / month
  • Route 53 (domain): ~$12 / year
  • CloudFront (optional): ~$1 – $3 / month

8) Transfer / Handover

  1. Create the client’s AWS account (or invite as IAM user if you built under your account).
  2. Create a new S3 bucket in client account and upload the site files.
  3. If using a custom domain: register/transfer to Route 53, then attach CloudFront for HTTPS.
  4. If SES/Stripe/APIs are used, recreate resources and update endpoints in index.html.
